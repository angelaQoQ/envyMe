# 1.响应式追踪变化原理

# 1.1 DOM刷新的机制

  • Vue异步更新队列的理解: Vue在更新DOM是异步执行的,只要侦听到数据变化,Vue会开启一个队列,缓冲在同一个事件循环中发生的所有数据变更. 如果同一个watcher被多次触发,只会推入队列1次, 可以在缓冲时去除重复数据可以避免没有意义的计算和DOM操作.在下一个事件循环tick中Vue刷新队列并执行已经驱虫的事件队列中的操作.也就是我们修改数据时,数据所在的组件并不会立即重新渲染,当刷新队列时,组件会在下一个tick中更新,需要在DOM更新后执行一些代码需要Vue.nextTick支持

1. 同一个数据多次修改,最后一次生效,即只会向事件队列中推入1次该数据的更新操作.
    组件在下一个[事件循环]tick中更新数据.

    组件内javascript代码:
    export default{
        data(){
            return {
                a:1
            }
        },
        watch:{
            // 1.1 监视变量a的改变, a修改时触发
            a(){
                console.log(this.a) // 1.2 打印1次: 5
            }
        },
        // 1.0 在初始化时 给a变量重复赋值,watch只会触发最后一次a的修改
        created(){
            this.a = 2  
            this.a = 3
            this.a = 4
            this.a = 5  
        }
    }

2. 如果希望在数据更新后立即调用某个函数callback,可以使用Vue.nextTick(callback) 进行插队操作(DOM更新结束就会调用callback)
3. 在Vue生命周期的created 钩子函数进行的DOM操作一定要放在Vue.nextTick()的回调函数中。

2. 当DOM需要随数据立即更新时,考虑使用this.$nextTick(callback)
