# 2.Vue混合策略

# 2.1 什么是'混入'

`mixin:使用mixins允许你封装一块在应用的其他组件中都可以使用的函数`

理解:
    1. mixins是一个JavaScript对象，可以包含组件中的任意选项，可以在mixin对象里定义组件的任何属性，在组件使用mixin时，mixin中的属性会添加到组件属性中

    2. 每次创建组件时，全局mixin都会添加到组件实例中，如果全局mixin和组件内mixin有同名的钩子函数，钩子函数都会执行，并且先执行先执行全局后执行组件内。

组件属性组成机制: 
    1. 实例的options加载顺序: 全局mixin ——> 组件内mixin ——> 组件options
    2. 钩子函数全都会触发, 触发顺序: 全局mixin钩子 ——> 局部mixin钩子 ——>  组件内钩子
    3. 同名函数会覆盖: 组件 > 局部混入 > 全局混入
    属性组成 类似于css的权重,与加载顺序有关.同名后加载生效

# 2.3 代码

  • "混入是对象, 组件属性它都有"
  • 需求: 让每个组件调用初始化检测登录状态
  1. 全局混入
    Vue.mix('myMixin' , {
        // 在初始化的时候检查用户的登录状态 token 
        created(){
            // 如果本地没有token,就跳转登录路由
            if(!window.sessionStorage.getItem('token')) return this.$router.push({name:'login'})
        }
    })

  1. 局部混入

    <template>
        <div class="container_mixin">
            {{ name }}
        </div>
    </template>

    <script>
        const myMixin = {
            // 在初始化的时候检查用户的登录状态 token 
            created(){ 
                // 如果本地没有token,就跳转登录路由
                if(!window.sessionStorage.getItem('token')) return this.$router.push({name:'login'})
            }
        }

        export default{
            mixins:[myMixin],
            data(){
                return {
                    name:'组件内局部混入'
                }
            }
        }
    </script>
