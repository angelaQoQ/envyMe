# 6.组件

所有的 Vue 组件同时也都是 Vue 的实例，所以可接受相同的选项对象 (除了一些根级特有的选项) 并提供相同的生命周期钩子

组件化,模块化的区别:

组件化:
    从UI角度,对 结构,功能,样式的封装,比如一个.vue组件
    如: Vue
模块化
    从功能的角度, 对 功能 的封装
    常见的有AMD规范(require.js) CMD规范(seajs) commonjs规范(node) ES6规范(javascript)

注意点:
1. 组件的模板中必须有且只有一个根标签

2. 组件中的data是一个方法，目的是让每一个组件维护一个自己的数据

3. 组件本质是vue实例

4. 组件文件名使用帕斯卡或者短横线,在引入使用时尽量使用短横线,避免与html标签冲突

# 1. 父组件函数 通过 自定义事件绑定给子组件标签 ,子组件调用传参

// 父组件
<template>
    <div class="container">
        // 子组件标签上通过自定义事件绑定 父组件的函数
        <son2 @Pmethod="pMethod"></son2>
        {{name}}
    </div>
</template>
<script>
    import son2 from "@/components/son2"
    export default {
        name:'parent_com',
        data(){
            return {
                name:'父组件初始数据'
            }
        },
        methods:{
            // 函数形参代表将来子组件调用时传入的数据
            pMethod( data_from_sonCom ){
                this.name = data_from_sonCom
            }
        },
        components:{
            son2
        }
    }
</script>

// 子组件
<template>
    <div class="son2">
        <button @click.native="changeParentData">点击修改父组件数据</button>
    </div>
</template>
<script>
    export default{
        name:'son2',
        data(){
            return {
                son_data1:'子组件数据'
            }
        },
        methods:{
            // 子组件的事件处理函数中
            changeParentData(){
                // 通过$emit触发父组件传入的函数
                this.$emit('Pmethod' , this.son_data1)
            }
        }
    }
</script>

# 3. 父组件 v-model实现子组件的双向数据绑定. 只适用于子组件中input触发传值

<子组件标签 v-model="父变量"> === <子组件标签 :value="父变量" @input="系统提供的事件名"> 
注意: 子组件中model设置prop(指定变量),event(指定触发事件)

    //父组件
    <template>
        <div class="container">
            <son4 v-model="parent_data"></son4>
        </div>
    </template>
    <script>
        import son4 from "@/components/son4"
        export default {
            name:'son4',
            data(){
                return {
                    parent_data:'父组件中的文字'
                }
            },
            components:{
                son4
            }
        }
    </script>

    // 子组件
    <template>
        <div class="son4">
            // 子组件必须使用表单元素接收和操作父组件 v-mode 传入值
            <input :value="parent_data">        
        </div>
    </template>
    <script>
        export default{
            model:{
                // 对应变量
                prop:'parent_data',
                // 触发修改的事件类型
                event:'input'
            }
        }
    
    </script>

# 6.3 组件缓存

# 使用场景

  • 台管理系统有个常见的需求: 从列表页跳入详情,从详情返回列表时不刷新不请求数据,即时加载.
  • 提供的解决方案:
1. 使用sessionStorage,数据存储获取都在浏览器本地,没有问题,但是操作繁琐
2. 通过地址栏记录详情页的id,页码等数据,但是需要在路由跳转时增加额外的代码
3. 使用Vue内置组件: <keep-alive></keep-alive> 配合 router缓存部分页面/组件

# keep-alive使用教程

# 1 使用 include/exclude属性

1. include: 字符串或正则表达式。只有匹配的组件会被缓存。
2. exclude: 字符串或正则表达式。任何匹配的组件都不会被缓存。

  • include="组件name|正则"
<script>
    export default{
        name:'keep-alive-com'
    }

<template>
    // 配合组件name属性

        <router-view></router-view>
    </keep-alive>

    <keep-alive :include="/ keep-alive-name , ... /">

    // 限制缓存的个数10个
    <keep-alive :max="10" include="keep-alive-name">

    </keep-alive>

</template>

# keep-alive 的注意点

<keep-alive>是Vue的内置组件，能在组件切换过程中将状态保留在内存中，防止重复渲染DOM

<keep-alive> 包裹动态组件时，会缓存不活动的组件实例，而不是销毁它们。和 <transition> 相似，

<keep-alive> 是一个抽象组件：它自身不会渲染一个 DOM 元素，也不会出现在父组件链中

# 防止组件缓存

  • 组件在路由切换时,vue-router为了提高效率会对同名组件重复使用时进行缓存,对应组件的钩子函数就不会再触发
    解决办法:
    1. 在router-view标签中设置唯一key  <router-view :key="$route.path"></router-view> , 可以是时间戳
    2. 在组件内对相关数据进行监视 watch:{}
    3. 在组件内路由守卫中对数据进行重新获取设置: beforeRouteEnter(){}

# 6.4 插槽

# 个人理解


第一次接触插槽的时候感觉很奇怪

个人理解就是: 
1. 买一个空的拉杆箱 
    ---- 创建一个带有插槽的子组件

2. 它的款式,大小,有几个轮子,拉杆多长由这个拉杆箱的生产厂决定的 
    ---- 这个子组件再被创建时就决定的结构,功能之类的

3. 这个箱子的里面出厂时是空的, 用的往里塞什么,它就装什么. 
    ---- 这个带插槽的子组件在被使用时 被传入了什么内容,它就在预留的位置展示什么内容

4. 如果生产厂家比较大气为了用着方便,可以在 出厂时在箱子里放了一些免费的衣服鞋子,用户在使用这个箱子时可以在箱子外部直接使用. 
    ---- 这些厂家提供的免费衣服鞋子就是 作用域插槽的初始数据了

# slot 作用域插槽

就像开头打的比方-4 : 厂自带内容  让  使用者拿着用的拉杆箱 就是 作用域插槽. 
在外部使用作用域插槽中的数据: template中用slot-scope 属性接受

----------------------------------------------------------------

// 需求: 父组件传给子组件数组, 子组件内遍历拼接父组件提供的结构

// 简单理解:  
    1. 父组件 给 子组件 一个数组
    2. 子组件 遍历 数组, 给slot标签绑定数据
    3. 父组件 template使用slot-scope接收 子组件返回的单独数据,接到结构中

----------------------------------------------------------------

// 子组件

<template>
    <div class="com-son">
        <ul>
            // 2. 子组件遍历数组,给slot绑定每条数据(等下父组件中要接受)
            <li v-for="item,index  in  parent_list" :key="index">
                <slot :item="item"></slot> //给父组件发送每条数据
            </li>
        </ul>
    </div>
</template>

<script>
    export default{
        // 子组件接收父组件数据
        props:["parent_list"]
    }
</script>

----------------------------------------------------------------

// 父组件
<template>
    <div class="container">

        <com-son :parent_list="list" > // 1. 父组件给子组件传个数组

            <template slot-scope="scope"> // 3. 父组件中 slot-scope属性值scope, 包含子组件给slot标签绑定的变量item
                <h5> {{scope.item.title}} </h5>
                <image :src="scope.item.imgSrc">

            </template>

            // 在插槽结构中使用时 推荐结构写法,不然一直scope点 scope点 scope点 ,太麻烦
            等价于 下面的写法
            <template slot-scope={item}>
                <h5> {{item.title}} </h5>
                <image :src="item.imgSrc">
            </template>

        </com-son>
    </div>
</template>
<script>
    export default{
        data(){
