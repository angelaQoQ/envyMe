# 3.2 数据 data

定义: 当前组件中携带的数据,当基本数据类型或简单的复杂数据类型 发生改变时Vue会主动更新视图
1. 在模板中使用直接使用变量名
2. 在methods中使用 需要this.变量名

<script>

    // 根组件中的data是对象
    const vm = new Vue({
        data:{
            str: '根组件数据是对象'
        }
    })

    // 单文件组件中的data是函数--为了保证每个组件实例的数据都是私有的
    export default{
        data(){
            return {
                str:'单文件组件数据是函数'
            }
        }
    }

注意: 

1. data中的变量一定要有初始值

2. 显示的视图更新
    数据为 复杂数据类型时,要合理使用this.$set(变量名,  对象属性或数组索引 , 值) 手动触发视图的更新
    '数组||对象':  Vue只能监听到pop push splice shift unshift reverse 这几个方法才能实现自动更新

3. data中的数据一般只负责当前组件和相关的父子组件的数据. 如:ajax请求的结构列表数据; 如果是公用范围大的数据尽量放在vuex 或 本地缓存中共享
</script>

  • 显示的视图更新
原因: 
1. Object.observe被废弃,Vue无法检测对象属性的添加或者删除;
2. 由于 Vue 会在初始化实例时对属性执行 getter/setter 转化，所以属性必须在 data 对象上存在才能让 Vue 将它转换为响应式的.
    也就是说: 在初始化结束后添加属性abc 代码: vm.abc = 123, 变量abc不是响应式
3. 复杂数据类型被Vue监听的是'地址值',基本数据类型被监听的是真实值. 复杂类型数据某一属性值发生改变时,其整体地址并未修改,所以Vue兼听不到,无法更新DOM.
4. 显示的删除使用this.$delete(对象,'属性')
    如:
    data(){
        return {
            list:{name:1}
        }
    }

   this.list上数据改变时能够被getter记录到wather的依赖中,list修改时setter触发当前实例的watcher,从而更新VDOM,然后VDOM会把DOM更新;
    但this.list数组中第一项的name值修改时,是当前wather监听不到的,如果更新需要使用this.$set(this.list , "name"  , "小明")

# 3.3 方法 methods

存放当前组件的方法

<script>
    export default{
        methods:{
            fn(){
                // 注意保留this指向,切记不要在methods中使用箭头函数声明 fn
                let that = this
            }
        }
    }

注意:
1. 方法声明不要使用箭头函数

2. 在方法内部使用事件处理函数,定时器,内置对象方法时一定要注意: this指向发生改变的问题.
    
3.  在模板中绑定的事件处理函数如果没有参数, 可以在函数声明事准备第一个形参作为事件对象e; 
    如果函数有自己的形参,那么在绑定时要添加一个形式参数$event表示对象提供者.
    事件触发者: $event.target 
    事件委托者: $evnet.currenTarget 

4.  模板中绑定事件可以写@click="fn" 也可以写@click="fn()"

</script>

# 3.4 过滤器 filter

过滤器,加工数据的作用.

<template>
    <div>
        {{time | formatDate}}
    </div>
</template>

<script>
    export default {
        data(){
            return {
                time:Date.now()
            }
        },
        filters:{
            // 过滤器名(需要处理的数据 , 其他参数...){ 
            //    return 加工过的数据
            // }
            formatDate(oldValue){
                return oldValue.toLocalString()
            }
        }
    }

注意点:

1.  管道符前面的变量或者数据是 后面过滤器函数的参数.

2.  过滤器可以串联 data|f1|f2|f3...

3.  项目中使用频率高的数据加工函数要封装成全局过滤器
    Vue.filter( 过滤器名 , 回调函数 )// 每次声明一个

</script>

# 3.5 计算属性 computed

计算属性: 本质是函数,用法是变量
使用场景: 实时获取一个或者多个变量 计算的结果时
体现:   多个data中的变量综合计算的结果

<template>
    <div class="container">
        <!-- ...动态修改计数count -->
        总价:    {{totalSum}}
    </div>
</template>

<script>
    export default{
        data(){
            return {
                //商品件数
                count:0,
                //商品单价
                price:100
            }
        },
        computed:{
            // total做变量直接使用
            totalSum(){
                return this.count*this.price + '元'
            }
        }
    }

    注意: 和 watch属性的区别

</script>

# 3.6 观察属性 watch

监视某个变量, 在变量值变化时触发响应的函数

注意: computed是依赖的变量之一发生变化时会重新计算; watch是某个变量修改时作什么工作.

使用场景: 节流+搜索框change事件,请求并渲染搜索列表
<template>
    <div class="contianer">
        <button @click="changeCount">点击修改count值</button>
    </div>
</template>
<script>
    export default {

        data(){
            return {
                count:0
            }
        },
        watch:{
            count( oldVal , newVal ){
                alert('count值被修改了:'+newVal)
            }
        }
    }
</script>

  • 被监视值修改时 立即触发操作
    因为 DOM更新机制: Vue为了性能, 当数据修改时 更新操作会被推入事件队列,并不是立即修改并触发watch中的回调.(在进阶1.响应式追踪变化原理详解)

    如果需要被监视值修改时 立即触发操作:
    this.$watch(
        this.obj , 
        function(newVal , oldVal){/*触发操作*/},
        {
            deep:true, // 深度监听数据变化
            immediate:true // 立即触发回调
        }
        )

# 3.7 指令 directive

全局自定义指令:
Vue.directive( 不带v-的指令名 , {
	bind(){
		只调用一次，指令第一次绑定到元素时调用
	},
	inserted( el ){
		当被绑定的元素插入到 DOM 中时自动触发
	},
	update(){
		虚拟DOM更新时触发
	},
	unbind(){
		只调用一次，指令与元素解绑时调
	}

} )

# 3.8 组件 component

组件就是一个包含了 结构(template) 功能(scirpt) 样式(style) 的独立vue实例, 从属于所在的父组件$children属性中

引入并声明为组件后,可以做普通标签在父组件template中直接使用.

<script>
    //全局组件
    Vue.component(组件名, {组件配置对象})

    //私有组件
    import com1 from '...com1.vue'
    import com2 from '...com2.vue'
    export default {
        components:{
            com1,
            com2
        }
    }
</script>
