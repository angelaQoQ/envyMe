# 4.Vue基本语法

# 4.1 双向数据绑定 v-model

  • 修饰符:
    • v-model.lazy把input事件触发修改为change事件触发.
    • v-model.numebr把文本框中值自动修改为数字
    • v-model.trim去除内容两端空格
Vue功能核心: 修改视图中内容Vue可以更新内存, 监听到内存数据改变时更新其他相关视图.

多用于收集表单数据

原理: 见Vue双向数据绑定原理

<template>
    <div class="container">
        <!-- 收集用户名 -->
        <input v-model="name" />
        <!-- 展示用户同步更新 -->
        <span> {{name}} </span>
    </div>
</template>


<script>
export default{
    data(){
        return {
            name:'小明'
        }
    }
}
</script>

# 4.3 绑变量 v-bind || :

给标签属性绑定变量或者变量的运算,或者函数的执行结果.

<template>
    <div class="container">
        <!-- 传入对象变量设置类名 -->
        <p v-bind:class="className" >文字内容</p>

        <!-- 传入基本数据 -->
        <p v-bind:item_id="id" >基本无解</p>
        <!-- 语法糖(简写) -->
        <p :item_id="id" >基本无解</p>

    </div>
</template>


<script>
export default{
    data(){
        classNames:{
            active:true,
            isShow:true,
            ...
        },
        id:0
    }
}
</script>

# 事件修饰符

对本次事件起修饰作用 , 可以连写多重修饰.
<template>
    <div class="container">

        <!-- 阻止冒泡.stop -->
        <a @click.stop="doThis"></a>

        <!-- 阻止默认事件 .prevent -->
        <form @submit.prevent="onSubmit"></form>

        <!-- 只能被触发一次 .once -->
        <a @click.once="doThis"></a>

        <!-- 只能通过绑定元素触发,冒泡无效 .self -->
        <a @click.self="doThis"></a>

        <!-- 捕获阶段触发 .capture -->
        <a @click.captrue="doThis"></a>

        <!-- 滚动时立即触发 移动端性能高 .passive -->
        <div class="container" @scroll.passive="doThis" ></div>

    </div>
</template>

# 循环v-for

根据数据循环创建复用性结构

场景: 多用于页面初始化或获取到数据时的页面结构创建

注意点: 为了保证循环结构在数据更新时的状态稳定, 要设置key属性.

<template>
    <div class="container">

        <!-- 根据首页数据,创建列表 -->
        <ul v-for="(item , index) in dataList " :key="item.id">
            <li> 商品名称:{{item.name}} </li>
            <li> 商品价格:{{item.price}} </li>
            <li> 商品数量:{{item.count}} </li>
        </ul>

    </div>
</template>


<script>
export default{
    data(){
        return {
            dataList:[
                {name:'老王吉' , price:9527 , count:3911 ,id:0},
                ...
            ]
        }
    }
}
</script>

# 显示/隐藏 v-show||v-if

1. 控制元素显示隐藏 值为true显示,false隐藏
2. 注意: v-show: 是通过display属性控制; v-if通过移除增加元素控控制
3. 性能上: 频繁切换显示隐藏 推荐使用v-show . 页面重绘比重排的性能损耗小很多.
4. v-if可以配合 v-else 做分支显示效果
<template>
    <div class="container">
        div.showBox
        <div v-show="isShow"></div>
        <button @click.native.stop="changeShowState"> {{isShowState}} </button>
    </div>
</template>


<script>
export default{
    data(){
        return {
            isShow:false
            isShowState:"显示",
            changeShowState:false
        }
    },
    methods:{
        changeShowState(){
            this.isShow = !this.isShow
            this.isShow ? this.isShowState="隐藏" : this.isShowState="显示"
        }
    }
}
</script>

# 首屏闪烁 v-cloak

<!-- 
    因为Vue项目首页内容是数据+模板拼接而成的,在首次展示页面时 模板内容还未被数据结构替换,显示出{{}}等不希望用户看到的代码结构
    所以 Vue给提供了v-cloak指令, 其值是一个起到隐藏作用的样式名, 在mounted函数调用后,该属性会被自动移除掉
 -->
 <div id="app" v-cloak="hideClass">
 
 </div>

 <script>
    let vm = new Vue({
        el:"#app"
    })
 </script>

# 单次渲染 v-once

  • 只渲染元素和组件一次。随后的重新渲染,元素/组件及其所有的子节点将被视为静态内容并跳过。这可以用于优化更新性能。
