# 5.实例的生命周期(钩子函数)

# 5.1 钩子函数

  • 在程序执行到某一个时间点会自动调用的函数.统称为钩子

# 5.2 创建前后


beforeCreate    创建前
created         创建后

定义: 组件内部函数,变量初始化完成前后自动调用

场景: 
    1.用于最早的获取组件数据,或者调用组件内方法请求后台数据等 初始化操作.
    2.beforeCreate 可以在此时加一些loading效果，在created时进行移除

这个阶段执行的内容:
判断对象参数有无el属性,如果有就继续执行, 如果没有就报错停止代码,直到这个vue实例调用了$mount(el)方法

注意:
    1. 实例参数中有template,就会按照template编译render函数
    2. 如果没有template参数,就会将外部HTML编译为模板
    template参数中的结构 要比'#app'中的结构优先级高
最后如果vue实例调用了render函数,最终生成的结构以render函数的参数为主.

模板相关属性方法的优先级顺序为:  vm.render(结构参数) > template参数选项 > "#app"指定结构

# 5.3 装载前后

beforeMount:    挂载DOM前,    在此之前内存中vm的el属性值是undefined,也就是不存在真实的DOM结构
mounted:        挂载DOM后     

场景: 
    1.mounted钩子中可以最早最完整的获取到真实的DOM结构, 使用第三方dom插件时(如Echarts),插件初始化操作在mounted中进行
    2.mounted 当需要操作dom的时候执行，可以配合$.nextTick 使用进行单一事件对数据的更新后更新dom

这个阶段执行内容:
根据"#app"指向,把vue创建的结构替换掉之前的DOM结构

# 5.4 更新前后

beforeUpdate    数据更新前
updated         数据更新后

当vue发现data中的数据发生了改变，会触发对应组件的重新渲染

场景: 
    1. updated 当数据更新需要做统一业务处理的时候使用
    2. 不要在update beforeUpdate修改不定数据，否则会引起死循环
    3. updated 监听data里的所有的数据

destoryed       实例销毁后  调用后实例的所有东西都会解绑定，所有的事件监听器会被移除，所有的子实例也会被销毁
