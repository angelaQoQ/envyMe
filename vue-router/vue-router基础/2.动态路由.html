# 2.动态路由

# 场景:

电商网站中 添加新商品的组件 和 商品详情编辑组件 从结构功能上几乎一样 如图:

# 使用:

  1. 设置路由对象时准备参数 {... , path:'/edit/:id' }, 注意参数写法 /:参数名
        1. 路由对象设置: { path:'/edit/:id' } ,
    1
  2. 如果: 组件一定会渲染内容,直接获取参数 请求 交互 就可以.
  3. 如果: 组件不一定由内容, 使用适当的钩子函数进行判断,如下面的代码.

# 代码:

  • js
router.js
    import VueRouter from 'vue-router'
    //设置动态路由
    export default{
        routes:[
            // 同一个组件根据不同参数决定 不同内容
            {
                // 动态路由的参数是 斜线: 后的值
                name:'/edit&add/:id',
                component:()=> import '@/component/EditAndAdd'
            }
        ]
    }

            }
        }
    }

# 问题

因为参数改变而组件没变,会导致操作如: '/detail/1' 至 '/detail/2' 这种路由跳转组件不重新加载. 导致created等钩子函数不触发,

  1. 可以监听$route的变换,判断路由参数是否修改
  2. 可以使用组件内路由守卫beforeRouteUpdate,获取最新数据更新
export default{
    name:'detail',
    created(){
        // 这里获取this.$route.params.id 发送ajax请求
        this.getData(this.$route.params.id)
    },
    1. // 使用监听
    watch:{
        // 防止组件不重载造成数据不更新
        $route(){
            this.getData(this.$route.params.id)
        }
    },

    2.//使用组件内路由守卫
    beforeRouteUpdate(to,from ,next){
        // 组件内路由守卫中的this是当前vm实例
        this.getData(to.params.id)
    },


    methods:{
        getData(id){
            // ajax请求
        }
    }
}
