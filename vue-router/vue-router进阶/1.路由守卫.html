# 1.路由守卫

# 1.1 vue-router执行机制

路由触发流程:

1> 用户点击router-link 或者 $router方法被调用 ,导航被触发

2> 在要离开的当前组件中触发 组件内的离开守卫 beforeRouteLeave

3> 调用全局beforeEach守卫,查询比对当前要跳转到的路由地址和组件

4>  A 情况: 如果出现组件重用: 调用缓存组件内 beforeRouteUpdate

    B 情况: 如果没有出现组件重用:  
        1 在全局beforeEach遍历routes后调用相关路由配置对象的 beforeEnter 

        2 解析异步路由组件

        3 调用 全局的beforeResolve守卫

        4 导航被确认

        5 调用全局afterEach

        6 触发DOM更新

        7 用创建好的组件实例调用 beforeRouterEnter守卫传给next的回调函数

# 1.2 几个钩子函数

# 1. 全局

- to 跳转到的路由对象
- from 从哪来的路由对象
- next 类似node中中间件`放行`功能的回调

##### `全局前置`
beforeEach(to,from,next){}            vuerouter循环遍历routes之前调用
beforeLeave(to , from , next){}       vuerouter全局匹配到对应路由对象后调用


##### `全局解析`
beforeResolve(to , from , next){}     在所有组件内守卫和异步路由组件被解析之后


##### `全局后置`
beforeEnter(to,from,next){}
    
export default new VueRouter({
    beforeEach(to , from , next){
        // 任何路由切换或修改时都会调用
        // 处理过守卫功能后记得放行
        next()
    },
    beforeLeave(to,from,next){}
})

# 3. 组件内钩子

当前组件被路由对象解析时调用

组件中中

- to 到哪去
- from 从哪来
- next 放行

beforeRouteEnter(to,from,next){}      ---------- 访问不到this
beforeRouteUpdate(to,from ,next){}    ---------- `在组件缓存时记得用它` 可以访问到组件this
beforeRouteLeave(to , from , next){}  ---------- 可以访问到组件this

export default ({
    name:'一个组件',
    data(){
        return{
            res:[]
        }
    },
    async created(){
        this.res = await this.ajax()
    },
    beforeRouteEnter(to , from , next){
        // 需要缓存的组件 加载的时候存储 当前的组件的名称和参数
        // 同名不同参的组件 不会重复调用它
        window.$routeName = to.name
        window.$routeParam = to.params.id
        next()
    },
    async beforeRouteUpdate(to , from , next){
        // 绑定this
        let _this = this
        // 如果记录的缓存组件和当前要跳转到的组件名称相同,但参数不同 重新请求数据
        if(window.$routeName&&
            window.$routeParam&&
            window.$routeName === to.name&&
            window.$routeParam !== to.params.id
            ){
            // 此时发请求更新数据
            this.res = await _this.ajax(...)
        }
        next()
    }
})
