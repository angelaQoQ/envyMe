# 3.后台权限动态路由设置

# 场景

  • 在后台管理系统中用户登录后,展示出当前用户相关权限列表(侧边栏可操作内容)

# 实现思路

  • 用户登录成功后回调中 请求 当前用户的权限列表
  • 加载主页组件时,其子组件侧边栏内容根据 权限列表动态生成.

# 准备工作

  1. vuex
  2. 路由对象追加方法this.$router.addRoutes
  3. 登录组件,权限菜单组件
  4. 本地存储,不知道为啥想到他总觉着会用到(可以替换Vuex的功能)

Vuex中存权限列表是为了首页菜单是为了 权限菜单 动态创建导航结构使用
在Router中添加权限列表是为了 用户操作时 切换组件使用

# 注意点

  1. router在new Vue的时候必须要加载,所以router最初就要存在. routes使用$router.addRoutes添加内容
  2. 后台响应的权限列表一定要在 权限菜单栏加载之前 添加到菜单栏数据中
    • Vuex可以实现
    • 首页created钩子中可以实现

# 实现方式

# 动态路由追加

  1. 在一开始给routes中准备固定路由对象如: + 登录页 + 登出页 + 404页面,

  2. 全部动态权限默认都不在routes中设置.

const router = new VueRouter({
    routes: [
        // 只准备固定路由
        {path:'/login',...},
        {path:'/logout',...},
        {path:'/404',...}
    ]
})

        v-if="item.meta.allow" 
        :to="item.path" 
        :key="item.path"
    > 
        {{item.path}} 
    </router-link>

// 在登录成功的回调中发送请求
    //let authList = await 发送权限请求
    let authList = [
        { path: '/a', meta: { allow: true } },
        { path: '/b', meta: { allow: true } },
        { path: '/c', meta: { allow: false } },
    ]
// 在vuex中更新最新权限列表数据
    this.$Store.commit('存储权限列表' , authList) 
// 动态添加权限列表到Router中
    this.$router.addRoutes(authList)
