# 1.ES5

# 1.1 ES5,ES2015(ES6)的取舍

  1. ES5大部分是对原生对象的扩展,使Object,Array,Function的功能更多更强
  2. ES5对IE9以上浏览器都已经支持

# 1.2 对象扩展

# Object.getPropertyOf(o)

  • 获取参数的原型对象, 等价于 o.proto

# Object.getOwnPropertyNames(o)

  • 获取对象o的自有属性列表

# Object.create(o,option)

  • 根据参数option的描述,创建对象o , 等价于 Object.defineProperties(o , option)

# Object.defineProperty(o, attr, option)

  • 根据option设置对象o的attr属性

# Object.defineProperties( o , options )

  • 根据options设置对象o的多个属性

# Object.seal(o)

  • 限制未来修改
  • 修改对象o,使其不能添加新属性,不能修改已有属性. 限制未来修改

# Object.freeze(o)

  • 限制未来修改
  • 修改对象o, 使其不可写,不能配置. 比Object.seal() 意义更严格.

# Object.isSealed(o)

  • 判断对象o是否被封印

# Object.isFrozen(o)

  • 判断对象o是否被冻结

# Object.keys(o)

  • 返回对象o所有可枚举属性组成的数组

# Object.values(o)

  • 返回对象o所有可枚举属性值组成的数组

# Object.prototype.isPrototypeOf(o)

  • 实例共享方法
  • 判断调用者是否是参数对象的实例

# Object.prototype.propertyIsEnumerable(attr)

  • 实例共享方法
  • 判断调用者的attr属性是否是可枚举的

# 1.3 数组扩展

# Array.isArray(ar)

  • 判断ar是否是数组

# Array.prototype.indexOf( arg , i ) || Array.prototype.lastIndexOf( arg , i )

  • 数组实例共享方法;
  • 判断调用者中从索引i开始,匹配到的值arg的索引. 找不到返回-1;
  • lastIndexOf是倒着匹配

# 数组实例新增的遍历操作

  • callback模板
    callback
    ( item , index , array )=>{}
1
2

  1. arr.every(callback) --- callback返回布尔值判断

    • 数组中每一项都满足callback判断,结果是true; 否则是false

  2. arr.some(callback) --- callback返回布尔值判断

    • 数组中有一项满足callback判断,结果就是true; 都不满足是false

  3. arr.map(callback) --- callback中对数组每一项进行加工

    • 数组中每一项经过callback处理,把加工过的项按顺序组成新数组返回

  4. arr.filter(callback) --- callback返回布尔值判断

    • 数组中每一项经过callback判断,把满足判断条件的项组成新数组返回

  5. arr.forEach(callback) --- callback中对数组每一项进行加工

    • 取出数组中每一项值,参与callback中业务逻辑

  6. arr.reduce(callback,startIndex) --- callback中对参数进行累加

    • callback参数至少2: 参数1:代表累加累减的初始数据;参数2:代表数组中每一项
        let arr = [1,2,3,4]
    let str = arr.reduce(function( baseItem , item ){
        return baseItem + item
    })
    str结果是 1+2+3+4===10
