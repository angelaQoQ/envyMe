# 5.小程序数据驱动思想:

  • 使用VDom映射wxml中的结构
  • 在setData调用时系统会比对新的VDom对象和原有VDom对象之间的差别,将差异更新应用到旧的DOM树上
  • 因为数据处理和数据展示是在两个线程中执行,setData()是异步函数和,第一个参数是数据对象,第二个参数是数据处理完成后的callback
  • 在使用setData()时,每次只修改最小单位的数据,每次操作量不要超过1024Kb,data默认值不要设置undefined

# 6.App

  • 宿主环境提供了App()构造,必须在app.js中调用执行,其他页面中可以使用getApp()获取小程序对象App
  • App包含了3个生命钩子: onLaunch onShow onHide
    • App打开方式很多,系统给onLaunch 和 onShow提供了参数option对象:
      • path: 打开小程序的页面
      • query: 打开小程序的连接参数
      • scene: 打开小程序的场景值
      • shareTicket: 分享
      • referrerInfo: 由公众号/其他小程序跳转来
      • referrerInfo.appId: 跳转来的小程序/公众号的appid
      • referrerInfo.extraData: 跳转来携带的参数
  • 全局数据:app.js中globalData.

# 7.页面

  • 页面需要配置在app.json的pages项中,否则不会被加载
  • 宿主环境提供了Page()构造器来注册小程序页面
  • Page包含5个生命钩子: onLoad onUnLoad onReady onShow onHide
    • 时间上: onLoad > onShow > onReady
    • onLoad 页面加载, 页面销毁前,只调用一次,页面切换复用会触发onShow
    • onShow 页面展示, 页面切换触发调用
    • onReady 页面渲染完成, 页面销毁之前,只调用一次
  • Page包含4个页面交互钩子:
    • onPullDownRefresh :需要app.json>window中 || page.json中配置enablePullDownRefresh:true, 在逻辑完成后可以调用stopPullDownRefresh()结束效果
    • onReachBottom :在app.json>window中 || page.json中配置onReachBottomDistance:值
    • onPageScroll :回调参数的scrollTop属性表示卷曲值
    • onShareAppMessage : 返回值是对象: title:转发的标题; path:要转发的页面路径
  • 页面栈:
    • 通过navigateTo推入页面,组成的前后历史关系叫页面栈
    • 页面栈最多10层, 满10层后不能继续推入
    • redirectTo会替换页面栈尾项
    • switchTab会清空之前的页面栈,之前页面栈中的页面会被销毁掉

# 8.组件(就是标签或者标签结构)

  • 使用import/include标签的src进行引入
  • 共有属性: hidden控制组件显示隐藏
  • 原生组件: map,video,canvas,picker这些
    • 都提供"context": wx.create***Context()
    • 原生组件的限制: 因为不是Exparser渲染体系下生成的,所以CSS样式对原生组件不适用,如overflow:hidden;transform:rotate(100)等不能操作地图API, 针对原生API限制,我们可以使用cover-view 和 cover-image这两个原生组件进行覆盖

# 9.API

  • wx.on*开头的接收回调函数
  • 其他方法参数多为对象, 至少包含3个方法: success,fail,complete
  • wx.get* 获取宿主环境数据
  • wx.set* 设置宿主环境数据

# 10.事件

# 事件类型

  • touchstart
  • touchmove
  • touchend
  • touchcancel
  • tap 按下马上离开
  • longpress 长按350ms以上
  • transitionend 动画结束触发
  • animationstart 动画开始
  • animationend 动画结束触发

# 事件对象

  • type: 事件类型
  • timeStamp: 打开页面到事件触发的毫秒数
  • target: 触发组件的属性集合 事件触发者
  • currentTarget: 当前组件的属性集合 事件委托者
  • touches: 触摸点数组

# 事件修饰符

  • bind: 普通绑定 不会阻止冒泡
  • catch: 普通绑定 阻止冒泡
  • captrue-bind: 捕获绑定
  • captrue-catch: 捕获绑定阻止冒泡
