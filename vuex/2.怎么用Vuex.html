# Store

  • store :仓库,所有数据和数据修改的存放容器.
  • 创建Vuex.Store的实例,注入Vue实例中
store.js中
    export default new Vuex.Store({
        // 存放数据
        // 存放数据修改方法
    })

main.js中
    import store from 'store'
    new Vue({
        el:'#app',
        ...
        store
    })

# 概览表

属性名 由数据组成 由函数组成 接收参数 使用方式
state 是 组件computed中return this.$store.state.变量
getters 是 1参数 state 组件computed中 return this.$store.getters.变量
mutations 是 2参数 state , newVal 组件事件处理函数中 this.$store.commit(mutations中函数, 新数据)
actions 是 2参数 context , newval 组件事件处理函数中 this.$store.dispath( actions中函数 , 新数据 )

# Store.state

  • state: 存放组件共享数据, 组件内可以通过this.$store.state.变量名获取数据
  • 在组件中使用computed属性接受 this.$store.state.变量 , 当store中数据修改时组件内会同步数据
store.js中
    new Vuex.Store({
        state:{
            documentTitle:'文章标题'
        }
    })
组件中使用
    export default {
        name:'com',
        data:{},
        computed:{
            title(){
                return this.$store.state.documentTitle
            }
        }
    }

# Store.getters

  • getters: store的计算属性.
  • getters中函数接收state做第一个参数,经过getters中函数处理过的变量会存储在getters中
store.js中
    export default new Vuex.Store({
        state:{
            name:'小明'
        },
        getters:{
            // 类似计算属性,加工state中的数据, 函数名做变量 存储在getters中
            chinaName(state){
                return "中国名字:"+state.name
            }
        }
    })

组件内
    export default{
        name:'com',
        data(){},
        computed:{
            name(){
                return this.$store.state.name
            },
            chinaname(){
                return this.$store.getters.chinaName
            }
        }
    }

# Store.mutations

  • mutations: 通过commit向state中同步修改数据
  • 直接给this.$store.state.变量赋值 是不能被追踪的
  • mutations中的函数接收state做第一个参数, payload(载荷)做第二个参数
store.js中
    export default new Vuex.Store({
        state:{
            title:'标题'
        },
        mutations:{
            setTitle(state , newTitle){
                state.title = newTitle
            }
        }
    })

组件中
    export default {
        name:'com',
        methods:{
            handleClick(){
                this.$store.commit('setTitle' , '吴彦祖')
            }
        }
    }

# Store.actions

  • actions: 异步提交mutations. 让mutations帮忙提交数据更改, 在组件内通过dispatch提交更改
  • actions 可以包含任意异步操作,如: 定时器,ajax请求
  • actions 内函数接收context作为第一个参数, context是store实例的备份,store的方法和属性context都有; 接收payload做第二个参数.
store.js
    export default new Vuex.Store({
        state:{
            name:'尼古拉斯'
        },
        mutations:{
            setName(state , newName){
                state.name = newName
            }
        },
        actions:{
            // 就是把mutations中的函数调用 使用异步处理 包装一下
            delaySetName(context,newName){
                setTimeOut(function(){
                    context.commit('setName' , newName)
                },1000)
            }
        }
    })

# 总结:

  1. state: 负责提供数据
  2. mutations: 负责修改数据
  3. actions: 负责异步触发mutations. 存数据还是由mutations做
  4. getters: 负责加工数据
